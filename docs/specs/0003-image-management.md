# 0003: イメージ管理

## 概要

ローカルにダウンロード済みのコンテナイメージの一覧表示、レジストリからの新規イメージ取得
（pull）、不要イメージの削除を行う機能。Docker Desktopの「Images」タブに相当する。

## ユーザー価値

- 手元にどのイメージがあるか、サイズはどれくらいかを一覧で把握できる。
- `wslc pull` を打たなくても、GUIからイメージ名/タグを指定して取得できる。
- 使わなくなったイメージを削除してディスク容量を回収できる。

## スコープ

- ローカルに存在するイメージの一覧表示。最低限、以下の情報を表示する。
  - イメージ名・タグ（名前やタグを持たない、いわゆる untagged イメージも一覧・
    削除の対象に含める）
  - イメージID
  - サイズ
  - 作成日時
- ユーザーが明示的に最新状態を再取得できる（一覧を開いたとき・任意のタイミングで、
  実際にローカルに存在するイメージ状態を取り直せる）。
- レジストリ（例: Docker Hub、`lscr.io` 等）からのイメージ取得（pull）。
  - イメージ名（+タグ）をユーザーが指定して取得を開始できる。
  - 取得の進行状況（進行中/完了/失敗）が分かる。
- イメージの削除。
  - 本仕様では、コンテナ（実行中・停止中を問わず）から参照されているイメージは
    削除不可とする。強制削除は非スコープ（将来検討）。
- イメージが1つも存在しない場合の空状態表示。

## 非スコープ

- イメージのビルド（Dockerfileからのビルド）は将来検討。
- イメージのタグ付け・レジストリへのpush。
- レジストリの検索UI（Docker Hub検索等）。本仕様では、ユーザーが取得したい
  イメージ名を把握している前提とする。
- プライベートレジストリの認証設定（将来検討。認証が必要なレジストリからのpullは
  本仕様のスコープ外）。
- レジストリのアローリスト等エンタープライズ向け機能（[docs/reference/wsl-containers-platform.md](../reference/wsl-containers-platform.md) 参照、対象外）。
- 参照中イメージの強制削除（本仕様では非対応。参照しているコンテナを削除してから
  イメージを削除する運用に限定する）。

## 前提条件

- [0000: 土台](0000-foundation-empty-window.md) が完了していること。
- [0001: コンテナ一覧・基本操作](0001-container-list-basic-operations.md) の完了は
  前提としない。イメージが「使用中」かどうかはコンテナランタイム側の参照情報を
  元に判断するため、コンテナ一覧画面の実装有無に依存しない。

## 受け入れ基準

- [ ] アプリのイメージ一覧画面を開くと、ローカルに存在するイメージ（untaggedを含む）が
      名前/タグ/ID/サイズ/作成日時とともに表示される。
- [ ] イメージが1つも存在しない場合、空状態であることが分かる表示になる。
- [ ] 一覧表示中に、ユーザーが最新状態への更新を指示すると、実際にローカルに
      存在するイメージ状態（他ツールやCLIからの変更を含む）が反映される。
- [ ] イメージ名（+タグ）を指定してpullを開始すると、取得が進行し、
      完了後に一覧に反映される。
- [ ] pullが失敗した場合（存在しないイメージ名、ネットワークエラー等）、
      ユーザーに分かる形でエラーが表示される。
- [ ] コンテナから参照されていないイメージを選択して削除すると、誤操作を防ぐ確認の
      上で削除され、一覧から消える。
- [ ] コンテナから参照されているイメージを削除しようとした場合、削除できない旨が
      利用者に伝わり、イメージは一覧に残ったままになる。
- [ ] 削除が（参照中である以外の理由、例: ランタイム不調・権限エラー等で）失敗した
      場合も、失敗した理由がユーザーに分かる形で表示される。
