# 0005: ボリューム管理

## 概要

コンテナが利用するデータを永続化するためのボリュームの一覧表示、作成、削除を行う機能。
Docker Desktopの「Volumes」タブが持つ機能のうち、一覧・作成・削除に絞ったサブセットを
提供する（詳細検査（inspect）、検索/フィルタ、複数選択削除等は本仕様には含まない）。

## ユーザー価値

- どのボリュームが存在し、どのコンテナから参照されているか（あるいは
  どこからも参照されていない未使用のものか）をターミナルを使わずに把握できる。
- 不要になったボリュームを削除してディスク容量を回収できる。
- 新しいコンテナ用に事前にボリュームを用意しておける。

## スコープ

- ボリュームの一覧表示。最低限、以下の情報を表示する。
  - ボリューム名
  - 作成日時
  - 使用状況（既存コンテナのいずれかから参照されている場合は「使用中」とし、
    参照しているコンテナ名を確認できる。どこからも参照されていない場合は
    「未使用」とする）
- ユーザーが明示的に最新状態を再取得できる（一覧を開いたとき・任意のタイミングで、
  実際のボリューム状態を取り直せる）。
- 新規ボリュームの作成（名前を指定して空のボリュームを作成する）。
- ボリュームの削除。本仕様では、既存コンテナのいずれかから参照されているボリュームは
  削除不可とする（強制削除は非スコープ、将来検討）。
- ボリュームが1つも存在しない場合の空状態表示。

## 非スコープ

- ボリュームの詳細検査（inspect相当、マウントポイントの詳細情報等）。
- ボリューム内のファイル内容をGUI上で閲覧・編集する機能（将来検討）。
- ボリュームのバックアップ・エクスポート/インポート、複製（clone）。
- 一覧の検索・フィルタリング、複数選択による一括削除。
- 特殊なドライバオプション（ネットワークボリューム等）を指定した作成。
  本仕様は、既定のローカルボリューム作成に限定する。
- 使用中ボリュームの強制削除（本仕様では非対応。参照しているコンテナを削除してから
  ボリュームを削除する運用に限定する）。

## 前提条件

- [0000: 土台](0000-foundation-empty-window.md) が完了していること。

## 受け入れ基準

- [ ] ボリューム一覧画面を開くと、存在するボリュームが名前/作成日時/使用状況とともに
      表示される。
- [ ] 使用中のボリュームについては、参照しているコンテナ名を確認できる。
- [ ] ボリュームが1つも存在しない場合、空状態であることが分かる表示になる。
- [ ] 一覧表示中に、ユーザーが最新状態への更新を指示すると、実際のボリューム状態
      （他ツールやCLIからの変更を含む）が反映される。
- [ ] 名前を指定して新規ボリュームを作成すると、一覧に反映される。
- [ ] 未使用のボリュームを選択して削除すると、誤操作を防ぐ確認の上で削除され、
      一覧から消える。
- [ ] コンテナから参照されているボリュームを削除しようとした場合、削除できない旨が
      利用者に伝わり、ボリュームは一覧に残ったままになる。
- [ ] 操作が（参照中である以外の理由で）失敗した場合も、エラーの理由がユーザーに伝わる。
