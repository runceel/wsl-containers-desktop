# 0002: コンテナログ表示

## 概要

選択したコンテナの標準出力・標準エラー出力ログを表示する機能。過去ログの閲覧に加え、
稼働中のコンテナについてはリアルタイムに追記されるログ（tail -f 相当）を表示できるようにする。
Docker Desktopの「Logs」タブに相当する。

## ユーザー価値

- コンテナが正常に動作しているか、エラーを起こしていないかをターミナルを使わずに確認できる。
- アプリ開発・デバッグ中に、コンテナ内アプリケーションの出力をリアルタイムに追いかけられる。
- 過去のログをスクロールして遡り、問題発生時の状況を後から確認できる。

## スコープ

- 一覧（[0001](0001-container-list-basic-operations.md)）からコンテナを選択し、
  そのコンテナのログを表示する画面（または領域）。
- コンテナランタイムが保持しており取得可能な過去ログをまとめて表示する
  （どこまで遡れるかはWSL Containers側の保持範囲に従う。アプリ側で意図的に
  範囲を絞り込むことはしない）。
- 稼働中のコンテナについては、新しく出力される行がリアルタイムに追記表示される。
- ログ表示の一時停止/再開（大量ログが流れ続ける際に読みやすくするため）。
  一時停止は画面への反映を止めるだけであり、一時停止中に出力されたログも
  再開後に順序を保って確認できる（一時停止中のログが失われることはない）。
- 表示中のログをクリアする（画面表示のみのクリア。コンテナ側の実際のログを
  削除するものではない）。クリア後も、以降に新しく出力されるログは
  引き続き表示される。
- ログが存在しない場合の空状態表示。

## 非スコープ

- ログの検索・フィルタリング（文字列検索、ログレベルによる絞り込み等）は将来検討。
- ログのエクスポート（ファイル保存）は将来検討。
- 複数コンテナのログを横断的に検索・表示する機能。
- ログの保持期間・ローテーション設定（WSL Containers側の挙動に従う）。

## 前提条件

- [0001: コンテナ一覧・基本操作](0001-container-list-basic-operations.md) が完了し、
  コンテナを選択できる状態になっていること。

## 受け入れ基準

- [ ] コンテナ一覧から任意のコンテナを選択し、ログ表示を開くと、
      そのコンテナの既存ログ（ランタイムが保持している範囲）が表示される。
- [ ] 稼働中のコンテナが新たに標準出力/標準エラーに書き込むと、
      表示中のログ画面に自動的に追記される。
- [ ] ログの表示を一時停止すると新規追記が画面に反映されなくなり、
      再開すると一時停止中に出力された分も含めて、順序を保って表示される。
- [ ] ログをクリアすると表示上のログが消え、以降に新しく出力されるログは
      引き続き表示される（コンテナ側の実際のログには影響しない）。
- [ ] ログが1件も存在しないコンテナを選択した場合、「ログがない」ことが
      分かる空状態の表示になる。これは、取得や追従に失敗した場合の
      エラー表示とは見分けがつく形にする。
- [ ] 停止済みのコンテナを選択した場合、既存ログは引き続き閲覧できる一方、
      新規のリアルタイム追記は発生しない状態であることが分かる。
- [ ] 対象コンテナが削除された場合、ログ表示側でもエラーで落ちず、
      「コンテナが存在しない」ことが分かるように表示される。
- [ ] ログの取得やリアルタイム追従が失敗した場合、失敗した旨がユーザーに
      分かる形で表示される（ログが単に無いだけの空状態とは区別できる）。
