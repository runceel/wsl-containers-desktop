# 0004: コンテナ詳細・exec

## 概要

個々のコンテナに関する詳細情報（ポートマッピング、環境変数、マウント構成等）を確認できる
画面と、稼働中のコンテナ内にシェルで接続（`exec`）してコマンドを実行できる機能を提供する。
Docker Desktopの「Inspect」タブおよびコンテナ内ターミナル接続機能に相当する。

## ユーザー価値

- コンテナがどう構成されているか（公開ポート、環境変数、マウントされているボリューム等）を
  ターミナルでインスペクトコマンドを叩かずに確認できる。
- コンテナ内部の状態を調査・デバッグしたいときに、GUIからシェルを開いて直接コマンドを
  実行できる。

## スコープ

- コンテナ一覧（[0001](0001-container-list-basic-operations.md)）から任意のコンテナを
  選択し、詳細情報画面を開ける。
- 詳細情報として最低限、以下を表示する。
  - ポートマッピング（ホスト側⇔コンテナ側）
  - 環境変数
  - マウント（ボリューム/バインドマウント）構成
  - コンテナの基本情報（ID、作成日時、使用イメージ、状態、実行コマンド/
    エントリポイント）
  - 接続しているネットワーク情報
  - 直近の実行状態（例: 正常終了/異常終了、終了コード等。稼働中コンテナに
    ついてはヘルスチェックが構成されていればその状態）
- 稼働中のコンテナに対して、シェル（コンテナ内の対話的なターミナルセッション）を
  開き、コマンドを連続して入力・実行し、その出力を画面上で確認できる。
  セッションは、ユーザーが閉じるまで維持される（作業ディレクトリや環境変数の
  変化など、コマンド実行間の状態が保持された対話的なターミナルとして機能する。
  毎回独立したコマンドを単発実行するだけの機能ではない）。
- シェルセッションを閉じる操作。
- 1コンテナにつき同時に開けるシェルセッションは1つまでとする。既にセッションが
  開いている状態で再度シェル接続を試みた場合は、既存のセッションを継続利用する
  （新規に別セッションを開始しない）。

## 非スコープ

- コンテナ設定の編集（環境変数やマウントの変更は、コンテナの再作成が必要になるため
  本仕様の対象外。将来、新規作成フローの一部として検討）。
- 複数のシェルセッションを同時に開くマルチタブ機能（将来検討。まずは1コンテナにつき
  1セッション）。
- CPU/メモリ使用量等、リソース使用状況の数値表示（瞬間値・履歴を問わず一切）。
  これは [0008: ダッシュボード](0008-dashboard-overview.md)の対象とし、
  本仕様では扱わない。
- ファイルのコンテナ⇔ホスト間コピー（`wslc cp` 相当のGUI操作）は将来検討。

## 前提条件

- [0001: コンテナ一覧・基本操作](0001-container-list-basic-operations.md) が完了し、
  コンテナを選択できる状態になっていること。

## 受け入れ基準

- [ ] コンテナ一覧から任意のコンテナを選択し、詳細画面を開くと、
      ポートマッピング・環境変数・マウント構成・接続ネットワーク・実行状態・
      基本情報が表示される。
- [ ] 稼働中のコンテナに対してシェル接続を開始すると、対話的にコマンドを
      連続して入力・実行でき、その標準出力/標準エラーが画面に表示される。
- [ ] シェルセッション内であるコマンドの実行結果（例: 作業ディレクトリの変更）が、
      同一セッション内の次のコマンド実行に引き継がれる（セッションを閉じるまで
      対話的な状態が維持される）。
- [ ] 既にシェルセッションが開いているコンテナに対して再度シェル接続を行うと、
      新しいセッションが作られるのではなく、既存のセッションに接続される。
- [ ] 停止中のコンテナに対してシェル接続を試みた場合、接続できない旨が
      ユーザーに分かる形で伝わる。
- [ ] シェルセッションを閉じると、接続が切断され、画面上もその状態が分かる。
- [ ] 詳細情報の取得やシェル接続に失敗した場合、エラーの理由がユーザーに伝わる。
