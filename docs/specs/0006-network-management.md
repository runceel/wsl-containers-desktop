# 0006: ネットワーク管理

## 概要

コンテナが接続するネットワークの一覧表示、作成、削除を行う機能。
Docker Desktopの「Networks」タブが持つ機能のうち、一覧・作成・削除に絞った
サブセットを提供する（既存コンテナのネットワークへの接続/切断変更、詳細検査
（inspect）等は本仕様には含まない）。

## ユーザー価値

- どのネットワークが存在し、使用状況（接続中のコンテナがあるかどうか）を
  把握できる。
- 複数コンテナを同じネットワークに参加させて通信させたい場合に備えて、
  事前にネットワークを作成しておける（実際にコンテナをそのネットワークに
  参加させる操作自体は、将来のコンテナ作成フローの一部として別途検討する）。
- 不要になった自作のネットワークを削除して整理できる。

## スコープ

- ネットワークの一覧表示。既定で用意されているシステムネットワーク
  （bridge, none等）と、ユーザーが作成したネットワークの両方を表示する。
  最低限、以下の情報を表示する。
  - ネットワーク名
  - ドライバ種別
  - 接続中のコンテナ数、または接続中コンテナの有無
  - 作成日時
  - システムネットワークかどうか（削除可否の判断に使う）
- ユーザーが明示的に最新状態を再取得できる（一覧を開いたとき・任意のタイミングで、
  実際のネットワーク状態を取り直せる）。
- 新規ネットワークの作成（名前を指定して既定のドライバでネットワークを作成する）。
- ネットワークの削除。本仕様では、以下は削除不可とする。
  - 既に何らかのコンテナが接続中のネットワーク
  - 既定で用意されているシステムネットワーク（bridge, none等）
- ネットワークが1つも存在しない場合の空状態表示。ただし、システムネットワークは
  常に一覧表示対象であるため、ここでの「空」は「ユーザー作成のネットワークが
  存在しない」ことを指す。

## 非スコープ

- 高度なネットワーク設定（サブネット指定、カスタムドライバ、IPアドレス範囲指定等）。
  本仕様は既定設定でのネットワーク作成・削除に限定する。
- 既存コンテナのネットワークへの接続・切断（[0001](0001-container-list-basic-operations.md)の
  非スコープにある「新規コンテナ作成」と合わせて将来検討）。
- ネットワークの詳細検査（inspect相当、接続中コンテナの一覧・IPアドレス等の詳細情報）。
- 一覧の検索・フィルタリング。

## 前提条件

- [0000: 土台](0000-foundation-empty-window.md) が完了していること。
- WSL Containersが、ネットワークの一覧・作成・削除、および作成日時・接続中
  コンテナ数の取得をサポートしていること（Public Preview段階のため、実装時に
  [docs/reference/wsl-containers-platform.md](../reference/wsl-containers-platform.md)
  で最新のAPI提供状況を確認すること）。

## 受け入れ基準

- [ ] ネットワーク一覧画面を開くと、システムネットワークとユーザー作成の
      ネットワークの両方が、名前/ドライバ種別/接続中コンテナ数/作成日時と
      ともに表示される。
- [ ] ユーザー作成のネットワークが1つも存在しない場合、その旨が分かる表示になる
      （システムネットワークの表示自体は消えない）。
- [ ] 一覧表示中に、ユーザーが最新状態への更新を指示すると、実際のネットワーク状態
      （他ツールやCLIからの変更を含む）が反映される。
- [ ] 名前を指定して新規ネットワークを作成すると、一覧に反映される。
- [ ] コンテナが接続していない、ユーザー作成のネットワークを選択して削除すると、
      誤操作を防ぐ確認の上で削除され、一覧から消える。
- [ ] コンテナが接続中のネットワークを削除しようとした場合、削除できない旨が
      利用者に伝わり、ネットワークは一覧に残ったままになる。
- [ ] システムネットワークに対しては、削除の操作自体が行えない、またはその場で
      削除できない旨が明示され、実際には削除されない。
- [ ] 操作が（接続中・システムネットワークである以外の理由で）失敗した場合も、
      エラーの理由がユーザーに伝わる。
