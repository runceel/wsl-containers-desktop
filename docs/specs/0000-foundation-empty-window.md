# 0000: 土台（空のWinUIウィンドウ起動）

## 概要

WSL Containers Desktopアプリの土台となる、最小構成のWinUI 3デスクトップアプリを用意する。
ウィンドウを起動すると、ナビゲーション用のシェル（今後追加される各画面を切り替えるための
枠組み）が表示されるが、この時点では中身の機能（コンテナ一覧等）は一切実装しない。
以降の機能仕様（0001〜）は、すべてこの土台の上に追加されていく前提とする。

## ユーザー価値

このマイルストーンの時点では、エンドユーザーがコンテナ管理を行える価値はまだない。
主な価値は「後続機能を小さく安全に積み上げられる土台を用意すること」自体にある。

- 機能がまだ何もない状態でも、アプリとして起動・終了ができることを早期に確認できる。
- 以降の機能追加が「土台に機能を差し込む」形で進むため、機能ごとの実装・レビュー・
  リリースのサイクルを小さく保てる（一度に大きな変更をレビューしなくて済む）。
- クリーンアーキテクチャの4層構成（[ADR-0005](../adr/0005-adopt-clean-architecture-layering.md)）
  が、後続の機能追加を受け入れられる状態として最初から成立していることを確認できる。

## スコープ

- WinUI 3アプリとして起動し、ウィンドウが表示され、正常に終了できる。
- 将来の主要機能（コンテナ、イメージ、ボリューム等）の画面を収容できるナビゲーションの
  枠組みが存在し、1つ以上のプレースホルダ遷移先（中身は空でよい）を切り替えられる。
- クリーンアーキテクチャの4層構成（[ADR-0005](../adr/0005-adopt-clean-architecture-layering.md)）
  を前提として、後続機能をどの層にも追加できる状態のアプリ基盤が存在する。
- ソリューション構成が [ADR-0006](../adr/0006-adopt-slnx-solution-file-format.md) に従い
  `.slnx` 形式で作成されている。

## 非スコープ

- コンテナ/イメージ/ボリューム/ネットワーク等、Docker Desktop相当の実機能（0001以降で対応）。
- WSL Containers（`wslc` / WSL Container API）との実際の通信。
- アプリのブランディング・最終的な見た目（アプリ名、アイコン、配色等）の作り込み。
- インストーラー・配布パッケージング（別途スコープとして検討）。

## 前提条件

なし（最初のマイルストーン）。

## 受け入れ基準

- [ ] クリーンな状態からチェックアウトしたソリューションをビルドでき、アプリを起動できる。
- [ ] アプリを起動すると、エラーなくウィンドウが表示される。
- [ ] ウィンドウを閉じると、アプリが正常終了する（プロセスが残留しない）。
- [ ] ナビゲーションの枠組みが存在し、1つ以上のプレースホルダ遷移先を選択して
      画面（中身は空でよい）を切り替えられる。
- [ ] ソリューション (`*.slnx`) を開き、Domain/Application/Infrastructure/Presentationの
      4プロジェクトが存在し、依存関係が [ADR-0005](../adr/0005-adopt-clean-architecture-layering.md)
      の向き（外側→内側）に従っていることを確認できる。
- [ ] WSL Containers/Docker相当の機能が何も実装されていない状態でも、上記の起動・
      画面遷移が問題なく成立する（＝後続実装の出発点として使える状態である）。
