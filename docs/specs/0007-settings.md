# 0007: 設定画面

## 概要

アプリ全体の設定を行う画面。初期版では、WSL Containers連携状態の確認と、
リソース制限（メモリ・CPU等、WSLのグローバル設定に対応する範囲のもの）の変更に
スコープを絞る。Docker Desktopの「Settings」画面が持つ機能の一部（リソース設定）に
相当するが、その全カテゴリをカバーするものではない。

## ユーザー価値

- WSLの設定ファイルを直接編集しなくても、アプリからリソース制限（メモリ量等）を
  調整できる。
- 現在利用しているWSL Containers環境が正しく有効になっているかを、GUI上で
  確認できる。

## スコープ

初期版で扱う設定カテゴリは以下の2つに限定する。

- WSL Containers連携状態の確認（変更は行わない）。
  - WSL Containers機能が有効かどうか、バージョン情報等の状態表示。
  - 動作要件（WSLの更新、pre-release要件等）を満たしていない場合、
    その旨が画面上で分かる。
- リソース制限設定の確認・変更（WSL Containers側で設定可能な範囲に限る）。
  - 例: 割り当てメモリ量、CPUコア数等、WSLのグローバル設定に対応する項目。
  - 設定変更の保存、保存後の値の永続化（画面を再度開いても変更後の値が
    表示される）、および反映結果（成功/失敗、再起動が必要な場合はその旨）の
    フィードバック。
  - 現在の設定値を既定値にリセットする操作。

## 非スコープ

- 利用するWSLディストロ/WSL Containers環境そのものの切り替え・選択変更
  （初期版は状態確認のみ。切り替え機能は将来検討）。
- ストレージ/ディスク関連設定、プロキシ設定、アプリの自動更新・通知設定、
  開発者向け設定等、リソース制限・WSL連携状態確認以外の設定カテゴリ全般。
- アプリの外観設定（テーマ、言語切り替え等）は将来検討。
- ネットワーキングモード（`consomme`等）やファイルシステム（`virtiofs`等）の詳細な
  切り替えUI。これらはWSL全体の設定に関わるため、本仕様では対象外とし、
  必要に応じて将来個別に検討する。
- GPU管理UI（対象外。[`docs/specs/README.md`](README.md)の初期スコープ方針を参照）。
- GPO/Intuneによるエンタープライズ管理機能、レジストリのアローリスト設定等、
  組織単位の制御・ポリシー編集機能（[docs/reference/wsl-containers-platform.md](../reference/wsl-containers-platform.md)
  参照。対象外）。

## 前提条件

- [0000: 土台](0000-foundation-empty-window.md) が完了していること。
- 対象環境がWSL Containersの動作要件（Public Preview、WSL 2.9.3以降の
  pre-release等）を満たしていること。満たしていない環境では、リソース制限の
  変更は行えず、状態確認画面でその旨が分かる表示になる。

## 受け入れ基準

- [ ] 設定画面を開くと、現在のWSL連携状態（有効/無効、バージョン等）が表示される。
- [ ] WSL Containersの動作要件を満たしていない環境では、その旨が画面上で分かり、
      リソース制限の変更操作は行えない。
- [ ] リソース制限（メモリ量、CPUコア数等）の現在値が表示され、変更して保存できる。
- [ ] 変更した設定を保存した後、設定画面を再度開くと、変更後の値が表示される
      （保存内容が永続化されている）。
- [ ] 変更した設定を保存すると、実際にWSL Containers側の挙動に反映される。
      反映に再起動等が必要な設定については、その旨が案内される。
- [ ] 設定をリセットすると、既定値に戻る。
- [ ] 設定の取得・保存に失敗した場合、エラーの理由がユーザーに伝わる。
