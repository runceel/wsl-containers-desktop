# 0009: 表形式一覧の操作性改善

## 概要

Containers、Images、Volumes、Networks、Dashboard stats の表形式一覧で、ユーザーが列幅を調整でき、
画面幅が狭い場合でも主要な行操作へ常にアクセスできる共通の操作体験を提供する。

## ユーザー価値

- 表示する情報やウィンドウ幅に合わせて列幅を調整し、必要な値を読み取りやすくできる。
- ページを移動して戻っても、同じ作業中は調整した列幅で一覧を確認し続けられる。
- 横幅が不足しても主要な操作が隠れず、コンテナの起動・停止等を迷わず実行できる。
- 対象ページ間で列幅変更と行操作の規則が統一され、画面ごとの操作方法を覚え直す必要がない。

## スコープ

- 次の表形式一覧で、ポインター操作とキーボード操作の両方による列幅変更を提供する。
  - Containers のコンテナ一覧
  - Images のイメージ一覧
  - Volumes のボリューム一覧
  - Networks のネットワーク一覧
  - Dashboard の稼働中コンテナStats一覧
- 列幅は一覧ごとに独立して保持し、同じアプリプロセス内のページ遷移・再表示後も維持する。
- アプリを終了して起動し直した場合、列幅を各一覧の既定値へ戻す。
- 行右端の主要アクション領域を列幅変更の対象から外し、ウィンドウ幅が不足しても表示・操作できるようにする。
- 各列には判読可能な最小幅を設け、列幅を0相当まで縮めて実質的に非表示にはできないようにする。
- 列境界を動かす場合は、拡張側で境界に接する列を広げ、縮小側にある全データ列を境界に近い順に
  最小幅まで縮める。拡張側に最大幅は設けず、縮小側の全列が最小幅に達した位置を可動限界とする。
  要求量が可動範囲を超える場合も操作全体を拒否せず、最も近い可動限界まで移動する。
  星幅で定義された列は列幅変更後も星幅を維持する。
- ユーザーが広げた列幅の合計が表示領域を超えた場合も、一覧全体を横スクロールさせず、
  主要アクション領域を右端に固定する。表示領域を超えたメタデータは切り詰め、
  ウィンドウを広げるか列幅を縮めると再び確認できるようにする。
- Containersでは、停止中のコンテナにPlayアイコン、実行中のコンテナにStopアイコンを同じ位置へ表示する。
  起動処理中・停止処理中・再起動処理中も対応するアイコンを同じ位置に残し、
  二重操作できない無効状態にする。
- Play、Stop、More actionsにはローカライズされたツールチップとアクセシブル名を設定し、
  列幅変更操作には隣接列を識別できるローカライズされたアクセシブル名を設定する。
- 現在のコンテナ状態モデルが持つ実行中・停止中の2状態を対象とし、将来新しい状態が追加された場合の
  操作規則は、その状態の仕様と合わせて別途定める。

## 非スコープ

- 一覧に表示する業務データや、既存の作成・削除・起動・停止等の処理内容の変更。
- 列の並べ替え、列の表示/非表示切り替え、一覧のソートやフィルタリング。
- 列幅の設定ファイル、ユーザー設定、クラウド等への永続化。
- 表形式一覧以外の画面レイアウトの全面刷新。
- 複数行に対する一括操作の追加。

## 受け入れ基準

- [ ] 対象となる5つの一覧すべてで、ポインター操作とキーボード操作の両方により列幅を変更できる。
- [ ] ある一覧の列幅を変更しても、他の一覧の列幅は変わらない。
- [ ] 列幅を変更して別ページへ移動し、同じ一覧へ戻ると、変更後の列幅が維持されている。
- [ ] アプリを終了して起動し直すと、各一覧の列幅が既定値へ戻る。
- [ ] 各列は判読可能な最小幅より狭くできず、列幅変更操作はキーボードフォーカスと
      隣接列を示すアクセシブル名を持ち、支援技術から識別できる。
- [ ] 列境界は、縮小側の隣接列が最小幅に達しても、同じ側の別のデータ列に縮小余地があれば
      移動を続け、縮小側の全列が最小幅に達した位置で停止する。拡張側に任意の最大幅は設けない。
      可動範囲を超える入力でも、境界は可動限界まで追従する。
- [ ] ウィンドウ幅が不足しても、行右端の主要アクション領域が表示され、メタデータがその領域へ重ならない。
- [ ] ユーザーが広げた列幅の合計が表示領域を超えても、主要アクション領域は右端に固定され、
      一覧全体に横スクロールは発生しない。表示しきれないメタデータだけが切り詰められる。
- [ ] 多数行で縦スクロールバーが表示されても、ヘッダーと各行の列境界およびアクション領域が揃っている。
- [ ] Containersでは停止中の行にPlay、実行中の行にStopが同じ位置へ表示され、対応する操作を実行できる。
- [ ] Containersの起動処理中はPlay、停止処理中または再起動処理中はStopが同じ位置に無効状態で残り、
      処理中にボタン位置が消失または反転しない。
- [ ] Play、Stop、More actionsには、表示言語に対応したツールチップとアクセシブル名がある。
- [ ] 列幅変更後も、一覧の更新、既存の行操作、空状態、エラー表示が従来どおり機能する。
