# 0001: コンテナ一覧・基本操作

## 概要

現在WSL Containers（`wslc` / WSL Container API）上に存在するコンテナの一覧を表示し、
起動・停止・再起動・削除といった基本操作をGUIから行えるようにする。
Docker Desktopの「Containers」タブに相当する、本アプリで最初に実装する実機能。

## ユーザー価値

- ターミナルで `wslc ps` 等のコマンドを打たなくても、稼働中/停止中のコンテナが
  一目で分かる。
- コンテナの起動/停止/再起動/削除といった日常的な操作をコマンドを覚えずにGUIから行える。
- 意図せず起動したままのコンテナ（リソースを消費し続けるもの）に気づき、
  簡単に止められる。

## スコープ

- コンテナの一覧表示。最低限、以下の情報を表示する。
  - コンテナ名
  - 使用イメージ名
  - 状態（実行中/停止中 等）
  - 作成日時
- ユーザーが明示的に最新状態を再取得できる（一覧を開いたとき・任意のタイミングで、
  実際のコンテナ状態を取り直せる）。
- 選択したコンテナに対する操作:
  - 起動
  - 停止
  - 再起動
  - 削除（本仕様では**停止中のコンテナのみ**削除可能とする。実行中のコンテナは
    先に停止しないと削除できない）
- 各操作は、対象コンテナがその操作を適用できる状態のときのみ実行できる
  （例: 既に実行中のコンテナに「起動」は適用できない）。適用できない操作は
  実行させない、またはその場で分かるエラーとする。
- 操作の成功/失敗が利用者に識別できる（誤操作を防ぐため、削除のような
  取り消せない操作の実行前には確認が行われる）。
- 操作が失敗した場合、対象コンテナの状態表示は操作前の実際の状態のまま保たれる
  （成功したかのように見えたり、古い状態のまま放置されたりしない）。
- コンテナが1つも存在しない場合の空状態表示。

## 非スコープ

- コンテナの詳細情報表示（ポートマッピング、環境変数、マウント等）→ 0004で対応。
- コンテナログの表示 → 0002で対応。
- コンテナへのシェル接続（exec） → 0004で対応。
- 新規コンテナの作成（イメージからの `run`）。本仕様は「既存コンテナの一覧・基本操作」に
  限定し、新規作成フローは別途仕様化する（将来検討）。
- イメージの管理（pull/削除等） → 0003で対応。
- 複数コンテナの一括操作（全選択して一括停止 等）。
- 一覧の検索・フィルタリング（名前・状態等での絞り込み）、ソート（将来検討）。
- 実行中コンテナの強制削除（本仕様では非対応。停止してから削除する運用に限定する）。

## 前提条件

- [0000: 土台](0000-foundation-empty-window.md) が完了し、ナビゲーションの枠組みが
  存在すること。

## 受け入れ基準

- [ ] アプリ起動後、コンテナ一覧画面に遷移すると、現在WSL Containers上に存在する
      コンテナがすべて表示される（コンテナ名/イメージ名/状態/作成日時を含む）。
- [ ] コンテナが1つも存在しない場合、一覧は空状態であることが分かる表示になる。
- [ ] 一覧表示中に、ユーザーが最新状態への更新を指示すると、実際の状態
      （他ツールやCLIからの変更を含む）が反映される。
- [ ] 停止中のコンテナを選択して「起動」を実行すると、コンテナが起動し、
      一覧上の状態表示が「実行中」に変わる。
- [ ] 実行中のコンテナを選択して「停止」を実行すると、コンテナが停止し、
      一覧上の状態表示が「停止中」に変わる。
- [ ] 実行中のコンテナを選択して「再起動」を実行すると、コンテナが再起動し、
      最終的に一覧上の状態表示が「実行中」になる。
- [ ] 既に実行中のコンテナに対して「起動」を、既に停止中のコンテナに対して
      「停止」を実行しようとした場合、その操作は実行されない、または
      分かりやすいエラーとして扱われる（無効な操作として一覧上の状態を
      壊さない）。
- [ ] 停止中のコンテナを選択して「削除」を実行すると、誤操作を防ぐ確認の上で
      コンテナが削除され、一覧から消える。
- [ ] 実行中のコンテナに対して削除を試みた場合、削除できない旨が利用者に伝わり、
      コンテナは一覧に残ったままになる。
- [ ] 操作が失敗した場合（例: WSL Containersが利用できない状態）、
      ユーザーに分かる形でエラーが表示され、対象コンテナの状態表示は
      操作前の実際の状態のまま変わらない。
