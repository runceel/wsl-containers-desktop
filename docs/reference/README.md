# 外部リファレンス (`docs/reference/`)

このディレクトリには、本プロジェクトが利用する**外部プラットフォーム・製品**に関する参照情報を置く。
`docs/adr/`（自分たちの意思決定）や `docs/design/`（自分たちの設計スナップショット）とは異なり、
ここに置くのは「外部の事実・仕様」であり、こちらが決定できるものではない。

## ADR / design との違い

| ディレクトリ | 内容 | 更新方針 |
|---|---|---|
| `docs/adr/` | 自分たちの意思決定 | 本文は不変。覆す場合は新規ADR追加 |
| `docs/design/` | 自分たちの設計の最新スナップショット | 上書き更新（経緯は書かない） |
| `docs/reference/` | **外部**プラットフォーム・製品の仕様・現状 | 外部の変化に合わせて上書き更新。**「最終確認日」を必ず明記** |

## 運用ルール

- 対象製品が **プレビュー版・開発中**の場合、仕様は頻繁に変わる。各ドキュメントの先頭に
  「最終確認日」と一次情報源（公式ブログ・公式ドキュメント）へのリンクを必ず書く。
- 実装や設計の判断に使う前に、可能であれば Microsoft Learn MCP
  （`microsoft-docs` plugin, 導入手順は [`AGENTS.md`](../../AGENTS.md) 参照）や公式ドキュメントで
  情報が古くなっていないか確認すること。
- ここに書く内容は「事実の要約」であり、「採用する/しない」という判断そのものは書かない
  （判断が必要な場合はADRを別途起票する）。
